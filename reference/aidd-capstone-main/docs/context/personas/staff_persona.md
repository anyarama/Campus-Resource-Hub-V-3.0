# Staff Persona

## Profile
- **Name:** Dr. Sarah Martinez
- **Role:** Faculty Member / Resource Owner
- **Department:** Engineering
- **Position:** Associate Professor

## Goals
1. Manage lab equipment and makerspace reservations
2. Approve student requests for restricted resources
3. Track resource usage and student engagement
4. Ensure fair access to limited resources

## Pain Points
- Time-consuming manual booking approvals
- No visibility into booking patterns
- Students sometimes misuse equipment
- Hard to balance accessibility with oversight

## Usage Patterns
- Checks approval queue 2-3 times per day
- Reviews booking requests within 24 hours
- Publishes new resources when equipment arrives
- Monitors resource reviews and feedback

## Technical Proficiency
- **Level:** Medium
- Comfortable with standard web tools
- Prefers simple, clear interfaces
- Values email notifications

## Quotes
> "I want to approve legitimate requests quickly, but I need to verify students are trained."

> "Seeing usage statistics helps me justify new equipment purchases."

## Success Metrics
- Can review and approve/reject bookings in under 1 minute
- Receives notifications for new booking requests
- Can view resource utilization reports
- Students provide context in booking requests
